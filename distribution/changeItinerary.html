Overview:

The change itinerary use case allows the customer or agent to search for an itinerary and potentially change it.

The change itinerary sequence is broken up into three diagram:

(A) List Itineraries: Customer selects 'change itinerary' from menu - the system presents a list of itineraries.
(B) Show Itinerary Details: Customer selects the itinerary to change - the system returns the selected itinerary which includes each segment of the itinerary.
(C) Remove Segment: Customer selects the segment to change. The system deletes the segment and the Prepare Itinerary use case is executed.

Assumptions:

  • User must be signed-on when the menu item for change itinerary is clicked.
  • Log-in use-case is executed if the user selects Change Itinerary from the main-menu if user is not signed on.
  • A paid itinerary cannot be changed.
