Overview:

The prepare itinerary use case allows the customer to search and reserve a seat. Seats can be reserved for round-trip or a one-way flight. The prepare itinerary sequence is divided into 5 diagrams:
(A) Search Flights : Customer enters flight search criteria, system responds with list of departure & return flights, customer selects flights, system responds with flights priced and alternative flights.

(B) Search Seats : Customer selects flights, system responds with available seats for the flight.

(C) Confirm Itinerary : Customer selects seats, system reponds with priced itinerary, customer confirms itinerary, system sends the priced itinerary to pay for itinerary use case.

(D) Confirm Itinerary Alt Flow 1(Customer not signed): Log-in use case is executed, Cusomter logs in with credentials, system responds with the itinerary confirmation window.

(E) Conform Itinerary Alt Flow 2 (Customer not a member): Create Profile use case is executed, user is signed on, system responds with the itinerary confirmation window.

Assumptions:

  • Customer cannot confirm the itinerary unless the customer is signed in.
  • If an anonymous user attempts to confirm the itinerary the log-in use-case is executed and the itinerary confirmation screen is displayed again: the customer will have to again confirm the itinerary.
  • The create profile use-case is executed after the log-in use case is executed, typically the log-in screen will also have a register user option.
