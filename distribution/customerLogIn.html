Overview:

This use case allows a customer/agent to sign on to the system via the web or the application client by providing a user name and password. Once the user name and password is validated against the authorization database, the system returns a customer id and other contextual information. This data is then retained in the user session which serves as the context and the sign-in token for other components to reuse.

Assumptions:

  • The password can be hashed using a suitable one-way well-known hashing algorithm.
